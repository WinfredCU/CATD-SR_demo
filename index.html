CATD-SR: Channel-Assisted and Codebook-Aware Token Denoising for Speech Restoration

Receiver-side neural-codec token restoration with selective editing and channel-likelihood posterior fusion.

Zewei Wang, Tat Ming Lok, and Yao Tang
Department of Information Engineering, The Chinese University of Hong Kong

Abstract

Neural audio codecs represent speech as compact residual vector quantization (RVQ) token streams. However, token-level corruption in these streams can severely degrade the final decoded waveform. CATD-SR studies a practical receiver-only restoration setting, where the transmitter, channel protection strategy, codec encoder, and codec decoder remain fixed, and only an add-on token restoration module is learned at the receiver.

The proposed system performs selective token denoising instead of full-token regeneration. It learns when to keep a received codec token and when to repair it, while predicting replacement-token distributions using codec-codebook information, cross-stage interaction, local temporal context, and stage-specific prediction heads. For structured valid-index replacement, CATD-SR further combines neural token logits with analytical channel likelihoods, so that the final repair decision is guided by both acoustic-token context and channel-law consistency.

This demo page focuses on listening examples. Numerical plots from the paper are intentionally omitted; each experiment subsection briefly states the corruption protocol and then presents the corresponding audio demos.

Proposed System

CATD-SR is a receiver-side add-on module placed between the fixed channel decoder and the fixed neural codec decoder. It does not redesign the transmitter, redundancy strategy, codec encoder, or codec decoder. The system restores the corrupted codec-token sequence first, and then lets the unchanged codec decoder synthesize the waveform.

Fig. 1. Step-by-step overview of CATD-SR. The receiver restores codec tokens before fixed codec decoding.
1

Receive corrupted codec tokens

The clean speech is represented by multi-stage RVQ acoustic tokens. After transmission and channel decoding, the receiver obtains a token sequence that may contain masked tokens, random replacements, burst errors, or structured bit-error replacements.

Novelty: The task is formulated directly in the codec-token domain under a strict receiver-only setting, rather than as waveform enhancement or end-to-end codec/transmitter redesign.
2

Build codebook-aware token representations

Each received token is embedded together with its RVQ-stage identity and codec-codebook information. Therefore, the model uses the structure of neural codec symbols instead of treating token indices as ordinary unrelated labels.

Contribution: Codebook-assisted representation connects discrete token prediction with the frozen codec embedding space, making the restoration module better matched to neural codec decoding.
3

Model cross-stage and temporal context

CATD-SR mixes information across RVQ stages at the same time frame and also models local temporal continuity along the token sequence. This helps the receiver infer whether a suspicious token is acoustically consistent with its neighboring stages and frames.

Contribution: The model exploits both RVQ cross-stage dependency and speech temporal continuity, which are not captured by independent token-wise correction.
4

Predict edit decisions and replacement tokens

The edit branch estimates whether each received token should be preserved or modified. The token branch predicts the most plausible clean token for positions that should be repaired. The final restored sequence is produced by selectively keeping reliable observations and replacing only harmful tokens.

Contribution: Selective editing avoids the over-modification problem of full-token regeneration and reduces the risk of breaking tokens that were already received correctly.
5

Fuse neural prediction with channel likelihood

For structured bit-error replacement, the corrupted token is still a valid codec index. CATD-SR computes an analytical likelihood for each candidate clean token according to its bit-level distance from the received token, and fuses this channel likelihood with the neural replacement logits.

Novelty: Posterior fusion combines a neural contextual prior with channel-law consistency, giving a MAP-style token decision without modifying the transmitter or channel code.
6

Decode restored tokens with the fixed codec

The restored token sequence is decoded by the unchanged neural codec decoder. The final objective is to improve the perceptual quality, intelligibility, and fidelity of the reconstructed speech waveform.

Contribution: CATD-SR remains modular and practical: it can be attached to an existing codec-token communication pipeline as a receiver-side restoration component.

Experimental Design and Audio Demos

Clean speech is first encoded into EnCodec acoustic tokens. Token-level corruption is then applied to the codec-token sequence. The corrupted tokens are either directly decoded, processed by benchmark restoration/enhancement methods, or repaired by CATD-SR before codec decoding. This section only includes experiment settings that are suitable for audio demonstration.

In the first four subsections, the demos compare clean reference, corrupted-token decoding, and CATD-SR restoration. The final structured bit-error subsection additionally includes benchmark demos to make the comparison easier to hear.

1. I.i.d. mask corruption

Independent token positions are replaced by a mask symbol. Because corrupted positions are explicitly indicated, this setting mainly evaluates whether CATD-SR can infer missing codec-token content from RVQ-stage and temporal context.

Independent corrupted positions are replaced by ordinary valid codec-token indices. This setting is more difficult than masking because a wrong token can still look legal. Consecutive codec frames are masked together, representing packet-loss-style damage. This setting tests whether temporal modeling can restore speech continuity when a local segment of the codec-token stream is missing.

Consecutive codec frames are replaced by valid but incorrect token indices. This setting combines burst damage and valid-index ambiguity: the receiver must infer both where the damaged segment occurs and what token content should be restored.

5. Structured bit-error replacement with benchmark comparison

Token indices are represented by binary bits, and independent bit flips produce valid replacement tokens. The received token is still a legal codec index, but its bit-level distance to each candidate token provides useful channel evidence.
